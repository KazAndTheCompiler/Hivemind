# DopaFlow packy_lorebook vs MemPalace — Comparison Report

## Architecture

### packy_lorebook (DopaFlow)
SQLite-based structured storage with two tables:
- `packy_lorebook` — session-level state: mood (JSON), active task, completed count, habit streak, focus minutes, review cards, journal flag
- `packy_lorebook_entries` — time-series snippet log of headlines and bodies

Also computes a `MomentumScore` (0–100) derived from live cross-domain queries (focus sessions, habit checkins, tasks, journal, nutrition).

**Storage model:** Structured key-value, not vector-based. No semantic search. Fast exact lookups, no similarity queries.

### MemPalace
ChromaDB-backed vector store with a layered architecture:
- **Wings** — person or project namespaces
- **Halls** — memory types: facts, events, discoveries, preferences, advice
- **Rooms** — specific topics; tunnels cross-reference rooms across wings
- **Closets** — summaries pointing to drawers; **Drawers** — raw verbatim content
- **Knowledge Graph** — temporal SQLite entity-relationship triples with validity windows

4-layer memory stack: L0 identity (~50 tokens), L1 critical facts (~120 tokens), L2 room recall, L3 deep semantic search.

**Storage model:** Raw verbatim vector embeddings, semantic similarity search, full conversation history.

## Use Cases Served

### packy_lorebook — optimized for real-time assistant context
- **Whispers:** proactive shell tips informed by current focus/habits/journal state
- **Answers:** ADHD-aware assistant replies using mood and active task context
- **Momentum tracking:** daily score + delta vs yesterday for gamification
- Latency-sensitive, session-scoped, structured

### MemPalace — optimized for long-term memory and retrieval
- **Semantic search:** "what did we decide about auth last month?"
- **Conversation recall:** full verbatim history, no summarization loss
- **Cross-project discovery:** tunnels find related topics across wings
- **Entity timeline:** what was Maya assigned to in January?

## Token Efficiency

**packy_lorebook** is extremely lightweight — a single row per session with scalar values. No embedding cost. Fast writes. But it stores no conversational context; it only stores derived state.

**MemPalace** stores raw exchanges — every message verbatim. Higher storage, but 96.6% R@5 recall on LongMemEval with zero API calls. The L0+L1 wake-up is ~170 tokens; searches fire on demand only. Better for long-horizon memory than packy_lorebook's snapshot approach.

## Recommendation

**They serve different purposes and should coexist — not replace each other.**

- **packy_lorebook** handles real-time session state for the Packy assistant — fast, structured, low-latency. Whisper logic depends on exact scalar values (focus_minutes_today, habit_streak, etc.) that MemPalace's vector store is not designed to track.
- **MemPalace** handles long-term conversation memory, semantic search, and cross-session recall — it would give Henrik the "remember everything" layer that packy_lorebook has no answer for.

**Proposed integration:**
- Keep packy_lorebook for Packy's real-time context (whispers, momentum, answers)
- Use MemPalace for the broader DopaFlow project wing to store conversation history, decisions, and preferences
- Emit Packy's lorebook events (whispers, momentum updates) into MemPalace via hall_events so they become searchable later

This gives DopaFlow both — real-time state tracking AND long-horizon semantic memory — without sacrificing either system's strengths.
