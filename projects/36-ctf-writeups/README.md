# Project 36: Write CTF writeups and methodology

## Overview

Guide to creating quality CTF writeups and methodology documentation.

## Why Writeups Matter

| Benefit | Description |
|---------|-------------|
| Learning | Teaching reinforces understanding |
| Portfolio | Demonstrates security skills |
| Community | Helps others learn |
| Documentation | Preserves solutions for future |

## Writeup Structure

```
1. Challenge Name
2. Category (Web, Crypto, RE, etc.)
3. Points / Difficulty
4. Description
5. Solution Walkthrough
   - Initial recon
   - Finding the vulnerability
   - Exploitation steps
   - Getting the flag
6. Key Takeaways
7. References
```

## Writing Tips

- Explain WHY, not just WHAT
- Include commands and output
- Show your thinking process
- Document dead ends too
- Add visuals/diagrams when helpful

## Tools for Writeups

- Markdown editors
- Draw.io for diagrams
- Carbon for code screenshots
- Pastebin for large outputs
